# Architectural Decision Records (ADRs)

This folder contains records of significant architectural and design decisions made for the ZoeGroups platform.

## Structure

### platform/
Decisions related to platform architecture, technology choices, and system design.

### curriculum/
Decisions related to curriculum structure, content design, and educational approaches.

## ADR Format

Each ADR should include:

**Title:** Short, descriptive title

**Status:** Proposed | Accepted | Deprecated | Superseded

**Context:** The issue motivating this decision

**Decision:** The change being proposed or decided

**Consequences:** What becomes easier or more difficult as a result

**Alternatives Considered:** Other options that were evaluated

## Naming Convention

`NNN-short-title.md`

Where NNN is a sequential number (001, 002, etc.)

Examples:
- `001-abp-framework-choice.md`
- `002-study-vs-course-model.md`
- `003-term-scheduling-approach.md`
